# 涂装双层滚动排产规则对齐设计

**日期**: 2026-03-23
**目标**: 让当前排产实现严格贴近 `doc/涂装双层滚动排产标准决策流程.md`，同时把库存滚动和结果展示做成可追溯的真实状态。

---

## 一、现状问题

- 现有算法流程名义上覆盖了短期需求、长期需求、风险、计划、阵型和库存更新，但关键约束未真正落地。
- 计算窗口没有严格使用用户输入的短期/长期分钟数。
- 长期计划没有按组进行前后平衡和车型产能上限控制。
- 阵型优化没有复用上一轮槽位。
- 库存更新没有把“更新库存”变成下一次排产的“当前库存”。
- 结果页面缺少读取区间、约束解释、库存更新明细等核对信息。

## 二、设计目标

- 以文档规则为最高优先级，算法与展示以规则对齐为准。
- 排产流程拆成可验证阶段，避免边算边写导致规则混杂。
- 当前库存读取口径改为：优先 `updated_quantity`，没有则读取 `current_quantity`。
- 本次排产完成后，将新库存同时回写到 `updated_quantity` 和 `current_quantity`，保证下一次计算直接使用更新后的库存。
- 为每次排产保存库存快照和阵型复用标记，保证历史结果可追溯。

## 三、算法分层

### 1. 上下文构建

- 读取产品、系统参数、安全库存、总成拉动。
- 读取涂装库存和注塑库存的有效当前值。
- 读取上一条已完成排产记录的阵型槽位，作为阵型复用参考。

### 2. 需求窗口计算

- 短期窗口按用户输入分钟数读取总成拉动前 N 条。
- 长期窗口从短期结束位置继续读取，长度按用户输入分钟数。
- 页面默认值仍可由系统推荐值生成，但一旦用户提交，提交值优先。
- 每条总成拉动拆成 `front` 与 `rear` 两个成品需求。

### 3. 风险计算

- 短期风险：`final_value = current_inventory - short_term_production_quantity`。
- 长期风险：`risk_value = safety_stock - final_value`。
- 长期按 `车型 + 颜色` 聚合，`group_risk_value = max(front_risk, rear_risk)`。

### 4. 计划生成

- 短期计划只处理 `final_value < 0` 的物料，按短期风险顺序止血。
- 长期计划按组风险值从高到低分配。
- 长期分配过程中执行：
  - 风险高的一侧优先分配。
  - `|front_count - rear_count| <= D`。
  - 同车型 `front + rear` 总车数不超过长期产能上限。
  - 总车数超限时高风险组优先保留。

### 5. 阵型优化

- 先尝试复用上一轮同槽位同产品。
- 不能复用的剩余计划按顺序填充空槽。
- 槽位数固定为一圈总车数，未使用槽位以空槽形式保留。

### 6. 库存滚动

- 涂装库存更新为：有效当前库存 - 本轮覆盖需求 + 本轮计划产出成品。
- 注塑库存更新为：有效当前库存 - 本轮计划投料数量。
- 更新结果写入：
  - `updated_quantity`
  - `current_quantity`
  - `update_time`
- 同时保存本次排产的库存快照，供历史详情展示。

## 四、数据变更

- `SchedulePlan` 增加说明字段，用于解释计划来源和约束结果。
- `FormationSlot` 增加 `is_reused`，标记是否复用上一轮槽位。
- 新增库存快照模型，保存每次排产的涂装/注塑库存前后值与变动量。

## 五、页面调整

- 计算页显示推荐窗口值与实际输入口径。
- 结果页补充：
  - 窗口读取区间
  - 计划说明
  - 阵型复用标记
  - 涂装/注塑库存更新表

## 六、验证重点

- 用户输入分钟数应直接影响需求窗口。
- 长期计划必须满足平衡约束和车型产能上限。
- 第二次排产必须读到第一次排产写回后的库存。
- 阵型复用应优先保留上一轮相同槽位产品。
